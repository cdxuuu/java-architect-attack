## 仓库概述

这是一个 Java 架构师面试每日练习知识库，采用"5天新题 + 1天串联 + 1天深挖"的训练节奏，按周组织内容。每周围绕分布式系统核心主题出题，用户作答后整理为问答和架构师梳理两个文档，并分析能力差距与补足方向。

已覆盖主题：
- **2026年05月第3周**：CAP 定理、分布式事务、分布式锁、缓存架构 | 串联整合 | 架构深挖-Raft协议
- **2026年05月第4周**：消息队列 | 进行中...

## 文件结构

按周组织，每周一个目录（如 `2026年05月第3周/`、`2026年05月第4周/`），目录内包含：
- `架构师面试-DayXX-主题.md` - 问答与练习整理
- `架构师面试-DayXX-主题-梳理.md` - 架构师角度梳理
- `架构师面试-Day06-串联整合.md` - 周六场景串联
- `架构师面试-Day07-架构深挖-XX.md` - 周日深挖某个点
- `架构师面试-能力差距梳理.md` - 当周能力差距分析

## 架构师面试每日练习
  训练节奏：5天新题 + 1天串联 + 1天深挖
  每周新建一个文件夹：以xx年xx月xx周开始，后续的计划都根据当前日期进入到具体的目录下进行
  格式：周一到周五每日文档分为两个，分别是问答和架构师角度梳理，周六叫场景串联，周日叫架构深挖，文档名字结尾带上日期
  已完成：查看当前年和周对应的文件夹里面的记录
  当日主题根据星期几决定：
  周一到周五：架构出题日和梳理日（要排除之前出过的题目）
  周六：串联日，将本周的新题和场景结合做串联，举出几个场景共我选择
  周日：以架构视角深挖本周的内容，训练我的架构师习惯
  每日的练习中需要指出我和架构师的差距，需要怎么去补足
  



## 内容特点

1. **结构化学习**：按天数组织内容，按周分目录，适合系统学习
2. **理论与实践结合**：每个主题包含原理讲解和实践要点
3. **知识梳理**：每个主题都有对应的"梳理"文档，便于复习
4. **面试导向**：内容针对架构师面试重点，注重高频考点
5. **能力差距分析**：每道题练习后指出与架构师水平的差距及补足方向
