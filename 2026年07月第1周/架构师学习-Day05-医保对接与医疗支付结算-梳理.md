## 一、国内医保体系全景

### 1.1 核心一句话
> **国内医保是国家强制社保，由"国家-省-市"三级经办，13.6 亿人参保，通过"医保信息平台"实现统一结算。医院 HIS 通过"医保接口模块"对接国家医保信息平台前端服务（HOS），走"身份核验 -> 费用上传 -> 结算 -> 撤销"四段式接口**
>
> 架构师必须能讲清：三级架构、三大基础数据库、三大目录、医保业务编码、HIS 与医保的边界、医保接口协议。

### 1.2 三级架构

```
国家医保局（NHSA）
  ↓ 政策 + 目录 + 跨省结算
省医保局 / 省医保中心
  ↓ 省级细则 + 省内异地结算
市医保经办（市医保中心）
  ↓ 协议 + 月度对账 + 拨付
定点医药机构（医院/药店/体检机构）
```

### 1.3 三大基础数据库

| 数据库 | 维护方 | 内容 | 作用 |
|---|---|---|---|
| **医保目录库** | 国家医保局 | 药品/诊疗项目/医用耗材 | 计费时按目录判定报销 |
| **定点机构库** | 国家/省医保局 | 全国 50w+ 定点医院/药店 | 跨省就医时校验 |
| **参保人员库** | 国家医保局 | 13.6 亿参保人 | 通过电子凭证查询参保状态 |

### 1.4 HIS 与医保的边界

```
HIS（医院侧）                医保系统（国家侧）
- EMPI                       - 参保人员库
- 门诊/住院业务              - 待遇享受判定
- 医生工作站                 - 医保目录
- 本地计费                   - 医保审核（规则引擎）
- 本地库存                   - 医保计算（报销比例/起付/封顶）
- EMR                        - 医保结算（统筹/个人账户）
                            - 医保基金监管
                            - 跨省异地就医
       ↑                          ↑
       └──── 专线 / VPN ──────────┘
        (HIS 医保接口模块)
```

> 架构师要点：HIS 与医保是两个独立系统，所有跨边界调用必须走标准接口（HOS 前端服务）。

## 二、医保三大目录与业务编码

### 2.1 三大目录

| 目录 | 内容 | 数量级 | 报销规则 |
|---|---|---|---|
| **药品目录** | 西药、中成药、中药饮片、谈判药 | 西药 ~1500，中成药 ~1400 | 甲类 100% 报销，乙类按比例自付后剩余报销 |
| **诊疗项目目录** | 检查、治疗、手术、康复 | ~5000 项 | 部分全额报销，部分按比例自付 |
| **医用耗材目录** | 高值/低值耗材 | ~3000 种 | 设定医保支付限价，超限自付 |

### 2.2 医保业务编码

| 编码类型 | 标准 | 示例 |
|---|---|---|
| 医保疾病诊断 | ICD-10（医保版） | A01.001 |
| 医保手术操作 | ICD-9-CM-3（医保版） | 01.01001 |
| 医保药品 | 国家药品代码（20 位） | XA01BD0010010000000X |
| 医保耗材 | 国家耗材编码（27 位） | 070101001000000000000000000 |
| 医疗服务项目 | 国家医疗服务项目（9 位） | 110100001 |

### 2.3 目录对照机制

```
医院本地目录                  医保目录
HM001234 阿司匹林    <--->    XA01BD00100100... (甲类)
HM001235 氯吡格雷    <--->    XA02BC00100100... (乙类自付 5%)
HM999999 进口保健品  <--->    (未对照)            (自费)
```

> 关键：未对照项目医保不报销，需患者自费。HIS 必须维护完整的本地-医保对照表。

## 四、医保结算"三段式"流程（必背）

```
┌────────────────────────────────────────────────┐
│ 阶段一：挂号/登记                                │
│ - 扫医保电子凭证                                │
│ - 调身份核验、待遇查询                           │
│ - 异地参保人调异地备案查询                       │
│ - 生成就诊号，标记"医保就诊"                     │
│ 【同步】                                        │
└────────────────────────────────────────────────┘
                  ↓
┌────────────────────────────────────────────────┐
│ 阶段二：就诊/计费                                │
│ - 医生开医嘱                                    │
│ - 实时调事前审核（同步）                         │
│ - 医嘱确认后计费                                │
│ - 上传费用明细（异步）                           │
└────────────────────────────────────────────────┘
                  ↓
┌────────────────────────────────────────────────┐
│ 阶段三：结算                                     │
│ - 调医保"结算"接口                              │
│ - 解析返回的结算单（总费用/自费/统筹/个人账户）  │
│ - 商业支付个人支付部分                           │
│ - 调医保"结算确认"接口                          │
│ - 生成发票                                      │
│ 【同步】                                        │
└────────────────────────────────────────────────┘
```

### 结算单字段（必背）

```
总费用 = 自费 + 自付 + 起付线 + 统筹支付 + 个人账户支付
       = 个人现金支付 + 医保支付

示例（公务员体检 500 元）：
- 总费用：500
- 自费：50（基因检测）
- 起付线：0（公务员体检不计起付线）
- 个人账户支付：450
- 个人现金支付：50
```

## 五、医保接口规范（HOS）

### 5.1 接口协议

```
协议：HTTPS + JSON / XML
签名：SM2 / RSA
加密：SM4 / AES（敏感字段）
编码：UTF-8
```

### 5.2 报文结构

```json
{
  "infno": "1101",          // 交易编号
  "infver": "1.0",          // 接口版本
  "infid": "uuid-1234",     // 唯一流水（幂等键）
  "orgno": "H12345678",    // 定点机构编号
  "input": { ... }          // 业务入参
}

响应：
{
  "infcode": 0,             // 0=成功，非0=失败
  "infmsg": "成功",
  "infid": "uuid-1234",     // 与请求 infid 一致
  "output": { ... }         // 业务出参
}
```

### 5.3 主要接口

| 接口类别 | 主要接口 |
|---|---|
| 医保电子凭证 | 电子凭证生成、电子凭证核验 |
| 身份核验 | 参保信息查询、待遇享受查询 |
| 费用结算 | 费用明细上传、结算、结算确认、结算撤销 |
| 医保审核 | 事前审核、事中审核、事后审核 |
| 异地就医 | 异地备案查询、异地就医结算 |
| DRG/DIP | 病案首页上传、DRG 入组、DIP 分值 |
| 目录管理 | 目录查询、目录对照 |
| 对账管理 | 对账文件下载、差错处理 |
| 医保电子票据 | 电子票据开具、电子票据查询 |

### 5.4 幂等设计

```sql
CREATE TABLE medical_insurance_idempotent (
  infid VARCHAR(64) PRIMARY KEY,         -- 唯一流水号（幂等键）
  infno VARCHAR(10),                     -- 交易编号
  biz_type VARCHAR(20),                  -- 业务类型
  biz_id VARCHAR(64),                    -- 业务 ID
  request_text TEXT,
  response_text TEXT,
  status VARCHAR(20),                    -- SUCCESS/FAIL/PENDING
  retry_count INT DEFAULT 0,
  UNIQUE KEY uk_biz (biz_type, biz_id)
);
```

> 与第5周支付幂等呼应：infid 是接口幂等键，业务唯一约束防止同一笔业务多次发起。

### 5.5 容错与重试

| 接口 | 容错 | 重试 |
|---|---|---|
| 身份核验 | 失败必须重试 | 指数退避 3 次（1s/2s/4s） |
| 事前审核 | 可降级（医生手工） | 不自动重试 |
| 费用上传 | 异步重试 | 5 次（1m/5m/30m/2h/12h） |
| 结算 | 必须重试 | 3 次（1s/2s/4s） |
| 结算确认 | 必须人工介入 | 不自动重试，告警 |
| 结算撤销 | 必须人工介入 | 不自动重试，告警 |

## 六、体检场景应用

### 6.1 四种付费模式判定

```
公卫覆盖人群（65+/慢病/孕产妇） -> FISCAL_ONLY 公卫免费
非公卫 + 公务员/慢病随访 + 余额充足 -> INSURANCE_PERSONAL 医保个人账户
非公卫 + 非医保范围 -> COMMERCIAL 商业自费
非公卫 + 医保范围 + 余额不足 -> MIXED 混合支付
```

### 6.2 套餐拆分示例

公务员套餐 500 元：

| 项目 | 类型 | 金额 |
|---|---|---|
| 身高体重/血压 | 医保甲类 | 0 |
| 血常规/尿常规/心电图 | 医保甲类 | 80 |
| 肝肾功/血脂/B超/CT | 医保乙类 | 370 |
| 基因检测 | 自费 | 50 |
| **合计** | | **500** |

拆分结果：
- 医保个人账户支付：450
- 个人现金支付：50

### 6.3 加项实时审核

```
方案 A：合并结算（推荐）
- 加项不立即结算，标记"加项未结算"
- 体检完成后统一结算

方案 B：分次结算
- 加项立即单独结算
- 缺点：两次结算单，两次发票

方案 C：撤销重结
- 撤销原结算 + 重新结算
- 缺点：撤销有风险
```

### 6.4 退款撤销

| 场景 | 能否撤销 | 处理 |
|---|---|---|
| 当日撤销 | 可 | 调"结算撤销"，自动冲正 |
| 当月撤销 | 可 | 调"结算撤销"，需医保审核 |
| 跨月撤销 | **不可** | 已拨付，走"医保退费"流程 |
| 已冲红发票 | 不可 | 发票已红冲，原结算不能撤销 |

## 七、互联网医院"互联网+医保"

### 7.1 允许的诊疗场景

| 场景 | 是否医保 | 备注 |
|---|---|---|
| 复诊（同医生 2 周内） | 是 | 必须复诊，首诊不能走医保 |
| 慢病随访（12 种慢病） | 是 | 高血压/糖尿病/冠心病等 |
| 处方流转 | 是 | 线上开方，定点药店取药 |
| 首诊 | 否 | 必须线下 |
| 急诊/手术/住院 | 否 | 必须线下 |

### 7.2 线上身份核验流程

```
T0: 用户在 APP 选择在线问诊
T1: 用户授权医保电子凭证（OAuth2 跳转医保电子凭证小程序）
T2: 后端调医保"线上身份核验"
T3: 判定是否医保范围（复诊/慢病随访）
T4: 问诊完成后医保结算（个人账户支付问诊费）
T5: 开方后药品配送（处方流转）
```

## 八、混合支付编排

### 8.1 三种支付方式优先级

| 优先级 | 支付方式 | 原因 |
|---|---|---|
| 1 | 医保个人账户 | 报销比例最高 |
| 2 | 商保直付 | 商保公司承担 |
| 3 | 微信/支付宝 | 用户自费兜底 |

### 8.2 金额拆分示例

```
订单：问诊 100 + 药品 300 + 配送 20 = 420 元

医保支付：250 元（问诊医保部分 50 + 药品医保部分 200）
商保直付：100 元（问诊商保部分 50 + 药品商保部分 50）
自费：70 元（配送 20 + 药品自费 50）

总计：250 + 100 + 70 = 420 元 ✓
```

### 8.3 Saga 补偿事务

```
执行顺序：
1. 调医保结算 -> 成功
2. 调商保直付 -> 成功
3. 调商业支付 -> 失败

回滚顺序（逆序）：
1. 撤销商保直付
2. 撤销医保结算

如果撤销也失败：
- 告警人工介入
- 不能丢数据
```

> 架构师要点：混合支付回滚是分布式事务问题，不能用 XA（医保/商保/支付是不同系统），必须用 Saga + 补偿 + 人工兜底。

## 九、商保直付

### 9.1 授权流程

```
T0: 用户选择商保直付
T1: 跳转商保 APP 授权（OAuth2）
T2: 商保返回授权码
T3: 后端调商保预审核（是否在保障范围）
T4: 商保返回预审核结果（可直付 X 元）
T5: 后端发起直付（商保直接付款给医院）
```

### 9.2 拒付兜底

```java
try {
    // 商保直付
    ciService.pay(order, amount);
} catch (CiRejectException e) {
    // 商保拒付 -> 转自费
    notifyUser("商保拒付：" + e.getReason());
    cpService.pay(order, amount);  // 调商业支付
}
```

## 十、三方对账

### 10.1 数据源

1. HIS 本地订单
2. 医保对账文件（T 日所有结算记录）
3. 商保对账文件（T 日所有直付记录）
4. 商业支付对账文件（微信/支付宝 T 日所有交易）

### 10.2 对账逻辑

```
Step 1: HIS vs 医保对账
  - 医保结算金额一致 ✓
  - HIS 有订单，医保无结算 -> 单边账
  - 医保有结算，HIS 无订单 -> 反向单边账

Step 2: HIS vs 商保对账
  - 商保直付金额一致 ✓
  - 商保拒付但 HIS 已发货 -> 需追回

Step 3: HIS vs 商业支付对账
  - 商业支付金额一致 ✓
  - 支付成功但 HIS 订单失败 -> 退款

Step 4: 三方汇总对账
  - HIS 总收入 = 医保支付 + 商保直付 + 商业支付
  - 总账平衡 ✓
```

### 10.3 差错处理

| 差错类型 | 处理 | 责任方 |
|---|---|---|
| HIS 有订单，医保无结算 | 调医保查询，补结算 | HIS |
| 医保有结算，HIS 无订单 | 调医保撤销，退款给医保 | 财务 |
| 商保拒付但已发货 | 联系用户补款 / 追回药品 | 客服+物流 |
| 商业支付多扣 | 退款给用户 | 商业支付运营 |
| 商业支付少扣 | 联系用户补款 | 客服 |
| 三方总账不平 | 人工核差 | 财务 |

### 10.4 拒付追回策略

| 配送状态 | 处理 |
|---|---|
| 已签收 | APP 推送 + 短信 + 3 日后电话催收 |
| 配送中 | 调物流召回 |
| 待发货 | 取消发货 |

## 十一、面试速记清单

### 必背知识点

1. **三级架构**：国家医保局 -> 省医保中心 -> 市医保经办 -> 定点机构
2. **三大数据库**：医保目录库、定点机构库、参保人员库
3. **三大目录**：药品目录、诊疗项目目录、医用耗材目录
4. **三段式结算**：挂号登记（同步）-> 就诊计费（事前同步+上传异步）-> 结算（同步）
5. **九大子模块**：身份核验/目录对照/事前审核/费用上传/结算/撤销/对账/异地/DRG
6. **接口协议**：HTTPS + JSON + SM2 签名 + SM4 加密
7. **幂等设计**：infid 唯一流水 + 业务唯一约束
8. **撤销限制**：当日/当月可撤销，跨月不可撤销
9. **互联网+医保**：复诊 + 慢病随访 + 处方流转
10. **混合支付优先级**：医保 -> 商保 -> 自费

### 必答场景题

- 公卫体检如何对接医保？-> 不对接（财政全额）
- 商业体检如何对接医保？-> 部分项目走个人账户（公务员/慢病随访）
- 加项如何处理？-> 合并结算（推荐）/ 分次结算 / 撤销重结
- 跨月退款怎么办？-> 走医保退费流程（不能撤销）
- 互联网医院如何走医保？-> 仅限复诊 + 慢病随访 + 处方流转
- 商保拒付但已发货？-> 已签收催收 / 配送中召回 / 待发货取消

## 十二、与第5周支付专题的对比

| 维度 | 商业支付（第5周） | 医保支付（本周 Day05） |
|---|---|---|
| 监管 | 央行/银保监 | 国家医保局 |
| 接口 | 微信/支付宝/银联 | 国家医保 HOS |
| 编码 | 商品 SKU | 医保业务编码（ICD/药品/耗材） |
| 报销规则 | 无 | 三大目录 + 起付线 + 报销比例 + 封顶线 |
| 结算 | 实时 | 三段式（挂号/计费/结算） |
| 对账 | T+1 双方对账 | T+1 三方对账（HIS+医保+商保） |
| 退款 | 自由退款 | 跨月不可撤销 |
| 幂等 | 商户订单号 | infid + 业务唯一约束 |
| 差错 | 长款/短款/单边账 | 单边账/拒付/已发货追回 |

> 架构师要点：商业支付与医保支付有共性（幂等/对账/差错），也有显著差异（医保有目录约束、报销规则、跨月限制）。能对比理解的架构师，能更好地驾驭医疗+支付的复合场景。
